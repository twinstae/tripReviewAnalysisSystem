금주목표

  • 데이터원 탐색 및 결정
  • Github를 이용한 협업 체계 정착
  • 데이터 수집 및 전처리 자동화 크롤러 프로토타입 제작.
  • 초기 데이터 확보

활동내용

  • 데이터원 결정
    • 각 팀원이 여러 사이트 탐색 후 보고서 작성.
    • 직접 (장소 : 서울 아현역 모 카페) 모여 회의를 통해 결정.
    • 페이스북, 인스타그램 등의 SNS는 크롤링이 어렵고, 유의미한 텍스트가 많지 않아 제외
    • 여행 후기 사이트 트립 어드바이저가 각 여행지별 후기를 다량 확보할 수 있어 1순위로 결정.
    • API를 제공하는 트위터와 댓글이 많은 유튜브를 2순위로 결정
    • 각종 여행사 사이트는 숙박과 항공권 관련 후기만 있어 3순위
  • 회의를 통해 각자 담당 분야에 따른 추가 학습 및, 개발 계획 지정
    • 참고 서적 구매 계획.
    • 1차 개발 목표는 프로토 타입 개발. 이번 주는 초기 데이터 확보에 중점.
    • 협업 툴 Github 이용하기로 결정.
  • Github를 이용한 공동 개발
    • 각 구성원이 Branch, Pull request를 이용한 코드 관리 방법, Markdown 사용법, 이슈 제기와 프로젝트 관리 방법 학습하였음.
    • 코로나 바이러스와 팀장 개인 사정(고향 방문)으로 구글 행아웃과 공동 시트를 이용해 화상회의 진행
    • https://github.com/twinstae/tripReviewAnalysisSystem/
  • 프로토 타입 크롤러 개발. 초기 데이터 확보
    • 머신러닝과 시각화를 위한 데이터 확보가 절실. 하지만 수동으로 복사 붙여넣기해서는 끝 없는 양.
    • 파이썬 프로그래밍 언어와 Selenium, Beautiful Soup라이브러리를 이용하여 웹에 데이터를 자동으로 수집해주는 crawler를 개발하였음.
    • 실제 크롤링 과정에서 다수의 버그 및, 개선 issue로 추가 개발 완료.
    • 최종적으로 총 180개의 여행지에 대한 각 최대 200개의 리뷰. 실제로는 20000 개 가량의 리뷰 데이터 확보.
